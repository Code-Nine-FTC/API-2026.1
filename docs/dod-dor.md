## 🚪 Definition of Ready (DoR)

Uma tarefa está pronta para ser iniciada quando:

* Possui descrição clara e objetiva
* Contém critérios de aceiteção definidos
* Está vinculada a um épico
* Possui Story Points atribuídos
* Possui tarefas e subtarefas definidas
* Dependências estão identificadas
* Dados necessários estão disponíveis ou planejados
* Está pequena o suficiente para ser concluída em uma sprint

---

## ✅ Definition of Done (DoD)

Uma tarefa (User Story) é considerada concluída quando:

* Funcionalidade implementada conforme requisitos
* Integrada com o backend e/ou banco de dados
* Testada e validada (mínimo: testes manuais)
* Sem erros críticos ou falhas bloqueantes
* Atende a todos os critérios de aceitação
* Documentação atualizada (ex: endpoints da API)
* Código organizado e funcional
* Pronta para uso no sistema (frontend ou API)
